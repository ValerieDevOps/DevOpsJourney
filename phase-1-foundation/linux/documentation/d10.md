# Day 10 Documentation – SSH Service & System Logging (Linux)

## Objective
Learn how to install, configure, start, monitor, and test the SSH service on Linux while exploring system logging with systemd-journald and journalctl.

## Environment
- Platform: Ubuntu 24.04 LTS (WSL2)
- Shell: Bash
- Services: OpenSSH Server, systemd-journald
- Tools Used: apt, systemctl, ssh, journalctl, script, mv, ls

## Activities Performed

### 1. Terminal Session Logging
Started terminal session recording using the script command to capture all terminal activities into a log file.

**Command**
```bash
script ~/DevOpsJourney/phase-1-foundation/linux/logs/d10.log
```

**Outcome**
- Successfully created a terminal session log.
- Recorded all commands and outputs for documentation purposes.

### 2. SSH Service Troubleshooting
Attempted to start the SSH service.

**Command**
```bash
sudo systemctl start ssh
```

**Result**
```text
Failed to start ssh.service: Unit ssh.service not found.
```

**Analysis**
The OpenSSH Server package was not installed, so the SSH service was unavailable.

### 3. System Package Update
Updated package repositories before installing new software.

**Command**
```bash
sudo apt update
```

**Outcome**
- Refreshed package indexes.
- Identified available system updates.

### 4. OpenSSH Server Installation
Installed OpenSSH Server.

**Command**
```bash
sudo apt install openssh-server
```

**Actions Completed**
- Installed OpenSSH Server.
- Installed OpenSSH SFTP Server.
- Generated SSH host keys (RSA, ECDSA, ED25519).
- Created SSH service configuration files.

**Outcome**
SSH service became available on the system.

### 5. Starting the SSH Service
Started the SSH daemon after installation.

**Command**
```bash
sudo systemctl start ssh
```

**Outcome**
SSH service started successfully.

### 6. Verifying Service Status
Confirmed that the SSH service was running.

**Command**
```bash
sudo systemctl status ssh
```

**Key Findings**
- Service status: Active (running)
- SSH daemon listening on port 22.
- Service managed successfully by systemd.

### 7. Monitoring SSH Logs
Monitored SSH activity in real time.

**Command**
```bash
sudo journalctl -u ssh -f
```

**Observed Events**
- Service startup events.
- Successful authentication attempts.
- Failed login attempts.
- Invalid user access attempts.
- Service stop and restart events.

### 8. SSH Login Testing
Tested SSH connectivity using localhost.

**Command**
```bash
ssh localhost
```

**Outcome**
- Successfully authenticated.
- Opened a new SSH session.
- Verified that the SSH service was functioning correctly.

### 9. Exploring systemd-journald
Inspected the Linux journal logging service.

**Command**
```bash
sudo systemctl status systemd-journald
```

**Observations**
- Service active and running.
- Responsible for collecting and managing system logs.
- Stores logs generated by system services and applications.

### 10. Checking Journal Storage Usage
Measured disk space consumed by system logs.

**Command**
```bash
sudo journalctl --disk-usage
```

**Result**
```text
Archived and active journals take up 366.9M in the file system.
```

**Learning**
System logs consume storage over time and should be monitored in production environments.

### 11. File Organization and Log Management
Moved generated terminal logs into the logging archive directory.

**Commands**

~/DevOpsJourney/phase-1-foundation/linux/logs/terminal-logging/d10.log

**Outcome**
- Organized terminal session logs.
- Maintained a structured logging directory.

# Linux Command Breakdown

## 1. stat -c "%a %n" *

This command displays file permissions and filenames for all items in the current directory.

### Command
```bash
stat -c "%a %n" *
```

### Breakdown

- `stat` → Displays detailed information about files and directories.
- `-c` → Allows a custom output format.
- `"%a %n"` → Format string:
  - `%a` → File permissions in **octal (numeric)** form (e.g., 644, 755)
  - `%n` → File or directory name
- `*` → Applies the command to all files and directories in the current directory.

---

## 2. find . -exec stat -c "%a %n" {} \;

This command recursively finds all files and directories and displays their permissions and names.

### Command
```bash
find . -exec stat -c "%a %n" {} \;
```

### Breakdown

- `find .` → Starts searching from the current directory (`.`) and recursively includes all subdirectories.
- `-exec` → Executes a command on every item found.
- `stat -c "%a %n"` → Runs `stat` with custom formatting:
  - `%a` → File permissions in octal form (e.g., 644, 755)
  - `%n` → File or directory name
- `{}` → Placeholder for each file or directory found by `find`
- `\;` → Marks the end of the `-exec` command  
  (The backslash prevents the shell from interpreting the semicolon)

---

## Summary

- `stat -c "%a %n" *` → Lists permissions for items in the current directory only.
- `find . -exec stat -c "%a %n" {} \;` → Recursively lists permissions for all files and directories.
