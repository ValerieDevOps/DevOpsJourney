## 4. Key Learnings

### 4.1 Linux Permission Model
- File access is controlled by:
  - owner permissions
  - group permissions
  - others permissions
- chmod 640 means:
  - owner: read/write
  - group: read only
  - others: no access

---

### 4.2 User vs Group Access
- Adding a user to a group does not automatically grant access unless file group ownership aligns
- Group changes must be applied using chgrp or usermod

---

### 4.3 sudo vs su
- su switches user and requires that user’s password
- sudo executes privileged commands using current user password

---

### 4.4 Debugging Approach
Effective troubleshooting sequence:
1. id → confirm user identity
2. groups → confirm permissions
3. ls -l → check ownership and file permissions
4. find → locate missing files
5. retry access with corrected path/permissions
