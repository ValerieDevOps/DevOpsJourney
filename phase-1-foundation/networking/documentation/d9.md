#### **Day 09 Documentation – Network Ports & Service Connectivity**

#### **Objective**

Learn how services listen on network ports, how to verify open ports, test service connectivity, and troubleshoot connection failures using Linux networking tools.

#### **Environment**

- Platform: Ubuntu (WSL2)
- Shell: Bash
- Tools Used: python3, http.server, ss, grep, curl, mv, ls
- Service: Python HTTP Server
- Test Ports: 8080, 9090

---

#### **Activities Performed**

#### **1. Created a Test Web Page**

Created a simple HTML file to be served by a local web server.

#### **Command**

```bash
echo "<h1>Port 8080 is alive</h1>" > index.html
```

#### **Verification**

```bash
cat index.html
```

#### **Output**

```html
<h1>Port 8080 is alive</h1>
```

---

#### **2. Organized Project Files**

Moved the HTML file into the ports project directory for testing.

#### **Command**

```bash
mv ~/devops-lab/index.html ~/devops-lab/ports/index.html
```

#### **Verification**

```bash
ls
```

Confirmed the ports directory existed within the project structure.

---

#### **3. Started a Local HTTP Service**

Used Python's built-in HTTP server to host the webpage on port 8080.

#### **Command**

```bash
cd ~/devops-lab/ports
python3 -m http.server 8080
```

#### **Output**

```text
Serving HTTP on 0.0.0.0 port 8080
```

#### **Observation**

The service successfully started and began listening for incoming connections on port 8080.

---

#### **4. Verified Open Ports**

Used the socket statistics utility to inspect active listening services.

#### **Command**

```bash
ss -tulpn
```

#### **Relevant Output**

```text
tcp LISTEN 0 5 0.0.0.0:8080 0.0.0.0:* users:(("python3",pid=6652,fd=3))
```

#### **Learning**

The output confirmed that the Python HTTP server process was actively listening on port 8080.

---

#### **5. Filtered Specific Port Information**

Used grep to isolate information about port 8080.

#### **Command**

```bash
ss -tulpn | grep 8080
```

#### **Output**

```text
tcp LISTEN 0 5 0.0.0.0:8080 0.0.0.0:* users:(("python3",pid=6652,fd=3))
```

#### **Observation**

Filtering makes it easier to identify which process owns a specific port.

---

#### **6. Tested Service Connectivity**

Connected to the running service using curl.

#### **Command**

```bash
curl http://localhost:8080
```

#### **Output**

```html
<h1>Port 8080 is alive</h1>
```

#### **Result**

Successfully reached the service and retrieved the hosted webpage.

---

#### **7. Simulated Connection Failure**

Attempted to connect to a port where no service was running.

#### **Command**

```bash
curl http://localhost:9090
```

#### **Output**

```text
curl: (7) Failed to connect to localhost port 9090: Couldn't connect to server
```

#### **Observation**

Port 9090 had no active service listening, causing the connection request to fail.

---

#### **8. Stopped the Service**

Terminated the running HTTP server.

#### **Action**

```text
Ctrl + C
```

#### **Output**

```text
Keyboard interrupt received, exiting.
```

#### **Result**

The service stopped and released port 8080.

---

#### **Errors Encountered and Resolutions**

| Issue | Cause | Resolution |
|---------|---------|---------|
| Connection refused on port 9090 | No service listening on port 9090 | Verified active ports and used correct service port |
| Incorrect HTML tag formatting | Typographical error in HTML heading tag | Corrected HTML structure |
| Command interruptions during terminal logging | Partial command entry during recording | Re-ran commands correctly |

---

#### **Key Concepts Learned**

#### **Network Port**

A logical communication endpoint used by services to receive network traffic.

#### **Listening Port**

A port actively waiting for incoming connections from clients.

#### **HTTP Server**

A service that receives HTTP requests and returns web content.

#### **curl**

A command-line client used to test connectivity and interact with web services.

#### **ss**

A Linux networking utility used to inspect sockets, ports, and active connections.

#### **grep**

A filtering utility used to search and isolate specific output from command results.

---

#### **Outcome**

Successfully created a web page, launched a Python HTTP service on port 8080, verified active listening ports using ss, tested successful connectivity using curl, simulated a connection failure on an unused port, and gained practical experience troubleshooting network services.

---

#### **Key Insight**

Services communicate through ports, and successful connectivity depends on both the service running and the correct port being used. Troubleshooting network issues begins by verifying that a process is listening on the expected port before investigating deeper application problems.
